# 实验席位排课系统

本地学生实验席位排课系统，面向高校实验课程管理员与助教，提供实验场次配置、席位管理、学生名单导入、手动排座、调换审批、签到记录和数据导出功能。所有数据本地持久化（SQLite），重启后数据完整一致，无需外部服务依赖。

## 技术栈

- **前端**：React 18 + TypeScript + Tailwind CSS 3 + Vite + Zustand + React Router
- **后端**：Express 4 + TypeScript（ESM）+ better-sqlite3
- **数据库**：SQLite，本地文件 `data/lab.db`

## 本地启动

### 前置条件

- Node.js >= 18
- npm >= 9

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

该命令会同时启动：
- **前端开发服务器**：http://localhost:5173 （Vite，带热更新）
- **后端 API 服务器**：http://localhost:3001 （nodemon，监听 `api/` 目录变化自动重启）

首次启动会自动在 `data/` 目录下创建 `lab.db` 数据库文件。

### 仅启动后端

```bash
npm run server:dev
```

### 仅启动前端

```bash
npm run client:dev
```

### 生产构建

```bash
npm run build
```

构建产物位于 `dist/`。

## 页面功能

### 1. 场次管理 `/`

- 卡片网格展示所有实验场次，显示场次名、日期、时间段、席位占用进度条、状态徽章
- "创建场次"弹窗：填写名称、日期、开始/结束时间、行列数，自动生成对应数量席位
- 点击卡片进入场次详情页
- 删除场次（已排座的场次会阻止删除，需先解除排座）

### 2. 名单管理 `/roster`

- **导入名单**：上传 CSV 文件，格式：`学号,姓名,班级,组别`（UTF-8 编码）
- **校验**：检测文件内重复学号及与已有名单学号冲突，有冲突则整批拒绝
- **样例下载**：点击"下载样例"获取可复现的 20 人样例名单 CSV
- **名单列表**：显示名单名、学生数、导入时间、使用状态（已关联场次标记为"使用中"）
- 行点击展开查看该名单下学生详情
- 已使用的名单禁止删除

### 3. 场次详情 `/session/:id`

顶部标签页切换：

| 标签 | 功能 |
|------|------|
| 席位视图 | 网格展示席位（绿色=空闲 / 蓝色=已占）。点击空闲席位分配学生，点击已占席位取消排座。顶部可绑定名单。 |
| 调换审批 | 卡片形式展示调换申请，显示双方学生、席位、申请理由。待审批显示"批准/拒绝"按钮，TA 审批需填写备注。已处理显示审批角色与备注。 |
| 签到记录 | 表格展示已排座学生签到状态。状态下拉切换（未签到/已签到/迟到/缺勤），批量签到按钮。 |
| 操作历史 | 时间线倒序展示本场次所有操作：排座、取消排座、调换申请、审批、签到、创建场次等，含操作人角色。 |

### 4. 导出 `/export`

三张卡片分别导出：
- **席位表**：列：学号、姓名、班级、组别、场次、席位号
- **考勤表**：列：学号、姓名、班级、组别、场次、签到状态、签到时间
- **操作日志**：列：操作时间、操作类型、操作人、详情

每张卡片可选择要导出的场次，点击"导出 CSV"下载文件（UTF-8 BOM，Excel 可直接打开）。

### 角色切换

页面右上角可切换"管理员 / 助教"角色，影响审批操作的 `approval_role` 记录。

## 数据持久化说明

所有数据存储于项目根目录的 `data/lab.db`（SQLite 数据库），包括：
- 场次配置（名称、日期、时间段、行列数、状态、关联名单）
- 席位定义（行列号、席位编号）
- 学生名单与学生信息
- 排座分配记录
- 调换申请（含理由、审批角色、审批备注、处理时间）
- 签到记录（状态、签到时间）
- 操作日志（操作类型、操作人、操作角色、详情、时间戳）

重启服务后所有数据保持一致，无需任何迁移或初始化脚本。

## 数据一致性保证

以下场景均通过事务和前置校验保证数据一致性：

1. **席位分配**：在同一数据库事务中检查席位是否空闲、学生是否已排座，通过后写入 assignment 并创建 attendance 记录
2. **调换审批**：批准前先比对申请时记录的双方席位与当前实际分配，不一致则返回 `SEAT_CHANGED` 错误，不做任何变更；**助教角色无审批权，审批直接返回 `TA_APPROVAL_FORBIDDEN` 错误**
3. **名单覆盖防护**：删除名单或覆盖同名名单前检查是否被场次引用，已使用则拒绝
4. **重复学号拒绝**：导入时检查文件内重复及数据库中同名单下学号冲突，任一存在则整批回滚
